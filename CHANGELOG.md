# 变更日志

本文档记录 `Schematica-survival` 的主要功能变更与行为调整。

## [0.1.4] - 2026-04-12
### 改动
- 现在玩家可以自行去设置里面调整打开投影gui的按键（默认是=键）
### 修复
- 现在玩家只能在任何界面都不打开的情况下才能够用按键打开投影的gui
- 修复了创建投影时输入名字会触发聊天框的BUG。

## [0.1.3] - 2026-04-02
### 新增
- 配置新增printer.clientProjectionUpload，若为false，则玩家无法用客户端数据打印建筑：
### 修复
- 修复打印机在服务器无法同步。
- 修复打印机在服务器各玩家看见的库存不一致。
- 修复打印机在服务器打印完后不卸载投影。

## [0.1.2] - 2026-03-31

### 新增
- 打印机投影选择流程新增“确认并加载（Confirm & Load）”统一入口：
  - 仅选择候选蓝图不会立即加载投影。
  - 点击确认并加载（或对应快捷键）后才会读取并显示投影。
- 打印机 GUI 与 `M` 键投影界面支持投影透明度调节（方块虚影与线框同步管理）。
- 打印机 GUI 新增按打印机坐标缓存配置（蓝图选择、旋转、镜像、最近一次应用签名）。
- 打印的位置有错误会有红框标出。

### 改动
- 打印机 GUI 关闭时会自动按当前设置尝试应用投影，并将当前配置持久化到下次打开。
- 旋转/镜像配置在 GUI 关闭并再次打开后保持不丢失。
- 材料“已存”统计统一以服务端打印机库存快照为准，减少客户端显示偏差。
- 打印、供料、撤回操作后会主动触发库存同步请求，材料面板刷新更及时。
- 投影渲染方式调整为蓝框+方块虚影。

### 修复
- 修复打印机旋转 90° 等配置“界面未加载但实际打印已生效”的状态不一致问题。
- 修复打印机 GUI 重开后旋转恢复为默认值的问题。
- 修复材料供给面板“已存”在打印/撤回后不及时更新的问题。
- 修复打印完成后选择状态与材料面板未清空的问题：
  - 打印成功（投影被自动卸载）后自动清空当前确认选择。
  - 打印失败或未完成时保留当前选择，避免误清空。

## [0.1.1] - 2026-03-30

### 新增
- 新增打印绿宝石成本规则：每 `printer.blocksPerEmerald` 个待放置方块需要 `1` 颗绿宝石（向上取整）。
- 新增打印机配置文件 `config/schematica_survival.properties`：
  - `printer.requireEmerald`：是否启用绿宝石成本（默认 `true`）。
  - `printer.blocksPerEmerald`：每颗绿宝石对应的方块数（默认 `32`）。

### 改动
- `M` 键投影与打印机投影流程解耦，避免互相撤回与状态串扰。
- 创造模式下，`M` 键投影不再被打印机逻辑禁用。

### 修复
- 修复打印机路径下潜在“无限供给”问题：材料检查与消耗统一按打印机库存执行。
- 修复打印机撤回后，破坏打印机时库存材料可能未掉落而消失的问题。

## [0.1.0] - 2026-03-30

### 新增
- 新增 `schematica_printer` 功能方块，并注册对应方块物品与合成配方（铁锭 + 红石）。
- 新增打印机命令：
  - `/schematica printer print <x> <y> <z> [replace|solid|nonair]`
  - `/schematica printer provide <x> <y> <z> <itemId> <subtype> [count]`
- 新增打印机 GUI 改造：
  - 增加 `Load Outline`（确认键加载投影）
  - 增加 `Undo`
  - 增加 `Rotate / MirrorX / MirrorZ`
  - 增加材料供给区域（每种材料一个按钮）
  - 增加可拖动进度条/滚动条，支持滚轮与方向键滚动
- 新增帮助命令行 `help.line9`，展示打印机相关指令。

### 改动
- 打印机打印不再直接消耗玩家背包材料，改为消耗打印机方块库存。
- 打印机方块升级为带 TileEntity 的容器方块，破坏时会掉落内部物品。
- 材料统计逻辑统一到通用库存路径（玩家背包与打印机库存共用检查/消耗流程）。
- 客户端命令别名匹配增加 `schematica printer` 路径。
- GUI 与命令提示文本进一步键化，补齐中英文语言键。

### 打印行为与安全规则
- 新增“冲突阻止打印”：
  - 当投影目标位置存在非空气方块，且与目标方块（含 meta）不一致时，打印会被拒绝。
  - 聊天提示会返回第一个冲突坐标与双方方块信息。
- 新增“同块跳过”：
  - 若世界方块与投影方块一致，则该位置不重复放置。
  - 该位置不计入材料需求/消耗。
- GUI 供给面板需求量会在投影加载后按当前世界状态重算，自动扣除已完成位置。
